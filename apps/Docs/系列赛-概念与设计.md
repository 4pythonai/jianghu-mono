# 系列赛 - 概念与设计

## 基本定义

「系列赛」是指两场或以上赛事的集合，这些赛事之间的数据具有关联性。系统需要按照设定的计算规则，把多场赛事的最终数据进行统计汇总，并以规范、通用、易懂的界面展示出来。

---

| 类型 | 说明 | 状态 |
|------|------|------|
| 系列队际赛 | 多场「队际赛」的集合，以球队为单位汇总成绩 | 本文档讨论 |
| 系列个人赛 | 多场赛事，统计个人跨场次成绩 | 待后续文档 |
| 其他类型 | 待定 | 待后续文档 |

## 系列队际赛

### 定义
系列队际赛是现有「队际赛」概念的集合。每一场子赛事就是一场现有的「队际赛」。

所有参赛队员归属于某一球队，打两场或以上的比赛，赛事进行中实时对球员成绩进行统计，按球队属性汇总，以球队为单位进行排名展示。

### 统计规则

创建系列赛时，由创建者选择使用以下两种统计规则之一：

| 规则 | 名称 | 说明 |
|------|------|------|
| A | 全局取前N名 | 每支球队所有场次的参赛球员（或组合）中，取成绩最好的前N名进行汇总，然后对球队排名 |
| B | 每场取前N名后汇总 | 每支球队在每一场次的参赛球员（或组合）中，取前N名进行汇总；若有X场，则共取 X×N 名球员（或组合）的成绩进行汇总，然后对球队排名 |

**N值设定**：由创建者针对每一场子赛事单独输入确认。

### 成绩统计规则

| 规则 | 说明 |
|------|------|
| 多场成绩计入 | 同一球员参加多场比赛，所有成绩均计入统计，不做去重 |
| 球队归属唯一 | 同一球员在同一系列赛中，不可代表不同球队 |

### 球队归属校验

| 项目 | 说明 |
|------|------|
| 校验时机 | 球员报名子赛事时即校验 |
| 冲突处理 | 发现冲突则阻止报名，或直接默认沿用上一场次代表的球队 |
| 校验范围 | 仅针对同一系列赛内的场次 |

## 核心概念

### 组织机构

「组织机构」是系列赛的主办方/组织者，独立于「球队」概念。

| 项目 | 说明 |
|------|------|
| 与球队关系 | 独立概念，不从属于球队 |
| 主要职能 | 作为赛事组织者，管理系列赛 |
| 权限控制 | 组织机构管理员可设定系列赛创建权限 |

### 创建权限

| 场景 | 权限 |
|------|------|
| 默认状态 | 任何人均可创建系列赛 |
| 组织机构限制 | 若组织机构管理员设定了创建权限，则需按其规则执行 |

---

## 创建方式

系列赛支持两种创建方式：

| 方式 | 说明 | 状态 |
|------|------|------|
| 先建后加 | 先创建系列赛，再添加子赛事 | 支持 |
| 后期并入 | 已有独立赛事符合整体逻辑时，可并入系列赛 | 待详细讨论 |

> 注：「独立赛事并入」涉及校验规则、N值处理等复杂问题，暂缓讨论。
