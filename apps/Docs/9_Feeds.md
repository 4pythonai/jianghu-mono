Feeds/game数据展示逻辑

# 定义
## 好友
   双方互相关注则互为好友

## 星标关注
   "我"的关注列表里面,某些人"我"对他们添加了"特别关注"标签,则就是我的"星标关注"



# 比赛类型
+ 普通比赛(即:小程序"普通创建"或“快捷创建”产生的)
+ 队内赛(即:小程序“更多创建”-"队内赛"创建产生的)
+ 队际赛(即:小程序“更多创建”-"队际赛"创建产生的)
+ 系列赛（即：小程序“更多创建”-“系列赛”创建产生的，包括“系列队际赛”“系列队内赛”“系列个人赛”和“莱德杯赛”）

   



# 星标球局
+ 我主动打了"星标"的比赛
+ 我创建的任何比赛
+ 星标好友的任何比赛
+ 我报名的任何比赛(取消报名会同时取消星标)

# 首页
## 我的
+ “我”参与的所有赛事（即实际参与打球的）
+ “我”未参与，但是“星标关注”对象参与打球的所有赛事（含系列赛，队际赛，队内赛）或球局。
+ “我”未参与，但是打了星标的所有赛事（含系列赛，队际赛，队内赛）或者“普通球局”。

 
api  Feed/myFeeds,参数:feed_type: "my" 



筛选:

     状态必须是[正在进行,已经结束的]里面的一种



排序规则:

如有重复,去重,然后:  
"我"正在进行的比赛排在最上面  
其他按照比赛创建时间进行倒排序

## 广场
+ "我"关注的所有人的赛事（含系列赛，队际赛，队内赛）,有密码也显示.
+ "我"的球队参与的任何赛事（含系列赛，队际赛，队内赛）
+ 系统推荐的任何赛事

筛选:

     状态必须是[正在进行,已经结束的]里面的一种


api  Feed/publicFeeds,参数:feed_type: "public" 


排序规则:

   如有重复,去重,然后  
   按照比赛创建时间进行倒排序

   用户可以对里面的每一项加上星标,加上后仍然在广场展示.



# 赛事 Feed
## 赛事招募
+ 所有的普通比赛(包含私密)
+ 所有的队内赛(包含非公开报名)
+ 所有的队际赛 (包含非公开报名)
+ 所有的系列赛



筛选:

     状态必须是[报名状态]里面的一种


api  Feed/myFeeds,参数:feed_type: "registering" 


 排序规则:以创建时间倒排序

   

## 已报名赛事
+ 所有我已经报名的队内赛
+ 所有我已经报名的队际赛
+ 所有我已经报名的系列赛

api  Feed/myFeeds,参数:feed_type: "registered" 

  
按照创建时间倒序排列
